Destructuring objects (2/2)

Het is van belang op te merken dat wanneer we x:x vervangen door x, het x: gedeelte wordt weglaten. Soms wil je een property van een object in de huidige scope een andere naam geven:

let { x:baz, y:bam, z:baz } = foo();
console.log(bam); // 4
console.log(x);   // reference error

De normale vorm target=source wordt bij destructuring omgedraaid. Bij x:bam is x de source (dat wat uit de functie komt) en bam is de target (datgene wat de waarde opvangt).

Object literals: target ← source
Object destructuring: source → target
