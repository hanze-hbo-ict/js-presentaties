Type-compatibiliteit

Typescript maakt gebruik van structural subtyping (static duck-typing). Dit is anders dan we gewend zijn in Java (maar lijkt wel weer op hoe het werkt in Python). Het volgende is valide:

  interface Named {
    name: string;
  }

  class Person {
    name: string;
  }

  let p: Named;
  p = new Person();  // OK, because of structural typing

In nominally-typed talen als C# of java zou dit niet kunnen, omdat Person geen subtype is van Named. In Typescript gaat dit wel goed, omdat de compilter ziet dat beide typen zich hetzelfde gedragen.

Er is over dit onderwerp een heleboel meer te melden. Check eventueel de documentatie.
